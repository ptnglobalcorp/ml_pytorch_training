# Contributing Guidelines

Thank you for your interest in contributing to the MLOps Training repository!

## How to Contribute

### Reporting Issues

If you find any errors, typos, or have suggestions for improvements, please open an issue.

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes following the code style guidelines
4. Commit your changes with a clear message
5. Push to your fork and submit a pull request

### Code Style

- Follow PEP 8 for Python code
- Use descriptive variable and function names
- Add comments for complex logic
- Include docstrings for functions and classes

### Documentation

- Keep documentation clear and concise
- Update README files when adding new examples
- Include code snippets with explanations

### Adding New Content

When adding new examples or exercises:

1. Place in the appropriate module directory
2. Include a README explaining the example
3. Add requirements.txt if dependencies are needed
4. Include solution code in the `solution/` subdirectory
